# N-of-M Findings Counter — Design Spec

**Issue:** #638
**Date:** 2026-04-20
**Status:** Approved

## Problem

The All Findings table shows a raw `· {filtered} of {total}` text in the `<h2>` heading (line 1281 of `report-app.jsx`), but it is always visible in the same muted style regardless of whether filters are active. Users have no clear visual signal that the displayed set is a filtered subset of the full registry.

## Decision

**Placement:** Option C — enhanced section heading.  
**Format:** `Showing X of Y` in a styled badge adjacent to the `<h2>`.  
**Unfiltered state (C1):** Hide the badge; show only the muted `· Y total` text.

## Behaviour

| State | Heading renders as |
|---|---|
| No filters, no search | `All findings · 60 total` (muted, existing style) |
| Any filter or search active | `All findings` + purple badge `Showing 12 of 60` |

The badge appears/disappears reactively as filters and search change — no user action needed.

## Implementation

### Where to change

**Single file:** `src/M365-Assess/assets/report-app.jsx`  
**Location:** `FindingsTable` component, the `<h2>` element (approx. line 1279).

**Build step required:** After editing the JSX, run `npm run build` from the repo root to recompile to `report-app.js`.

### Logic

`FindingsTable` already has:
- `filtered` — memoized array of visible findings (`useMemo` over `filters` + `search`)
- `filters` — prop from App state
- `search` — prop from App state

Derive a boolean `isFiltered`:

```jsx
const isFiltered = search.length > 0
  || filters.status.length > 0
  || filters.severity.length > 0
  || filters.framework.length > 0
  || filters.domain.length > 0
  || (filters.profile || []).length > 0;
```

### JSX change

Replace the current heading:

```jsx
// BEFORE
<h2>All findings <span style={{fontWeight:400,color:'var(--muted)',fontSize:13}}>· {filtered.length} of {FINDINGS.length}</span></h2>
```

With:

```jsx
// AFTER
<h2>
  All findings
  {isFiltered
    ? <span style={{
        marginLeft: 8,
        fontSize: 12,
        fontWeight: 500,
        background: 'var(--accent-soft)',
        border: '1px solid var(--accent-border)',
        color: 'var(--accent-text)',
        borderRadius: 20,
        padding: '2px 10px',
        verticalAlign: 'middle'
      }}>
        Showing {filtered.length} of {FINDINGS.length}
      </span>
    : <span style={{fontWeight:400, color:'var(--muted)', fontSize:13}}>
        · {FINDINGS.length} total
      </span>
  }
</h2>
```

### CSS variables used

These already exist in `report-themes.css` and `report-shell.css` for all four themes:

| Variable | Purpose |
|---|---|
| `--accent-soft` | Badge background (low-opacity accent fill) |
| `--accent-border` | Badge border colour |
| `--accent-text` | Badge text colour |
| `--muted` | Unfiltered `· Y total` text |

## Testing

1. Open `docs/sample-report/_Example-Report.html` in a browser — counter badge must NOT appear by default.
2. Apply any filter (e.g., click "Fail") — badge appears with correct `Showing X of Y` count.
3. Clear all filters — badge disappears, `· Y total` text returns.
4. Combine search + status filter — count reflects both constraints.
5. Verify badge renders correctly in all four themes (Neon, Console, Light, HighContrast).
